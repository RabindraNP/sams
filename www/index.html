Installation

Please read the following before you run SAMS:
  1. Although this version seems to be quite stable, it's still considered under beta testing. Thank you for helping us with your feedback.
  2. A subdirectory .sams/ will be created in your home directory (Z:\ under Windows in CSTARS network).  SAMS will store a number of supporting files there (configuration, preferences, database definitions, etc.). Do not make any manual modification to it. (Of course, you can delete it if you decide to stop using SAMS.)
  3. If you have run a prior version of SAMS, you may experience some compatibility problems. If you have worked on some databases, maybe you will need to recreate them.
  4. Your original spectral files (i.e. spectra files, Envi files, etc.) are not modified in any way (SAMS will only read your files to import signatures in a proper working space).
  5. See below for Known bugs and limitations .
Installation instructions
  • Requirement: A Java Runtime Environment (version 1.4 or newer) is required on your system since SAMS is a Java application..
  • Download the SAMS installer: install-sams-2.0r7.jar (updated 11/13/02, 1:40pm)
    Note: Use Shift-click or "Save link as..."
  • Run the installer by double-clicking on the installer file.
  • Follow the instructions.
CSTARS note: SAMS is ready to run at CSTARS lab on windows machines: From the Start menu, choose: All programs -> SAMS -> sams.jar

Recent changes

2.0r7 (11/13/02)
  • "Rename" action is now working. 
  • New "Change wavelength unit" operation that will apply a conversion factor to wavelengths. For example, if wavelength values are measured in nanometers, then a conversion factor of 0.001 will convert them to microns.
  • Selected->Compute  operations can now be applied on the current selection of groups. Only "many-to-one" operations can be applied. The operation is applied on a group-by-group basis. On each group, the operation is performed on its signature members and the resulting name signature is composed from the name of the group and the suffix given in the dialog. The group selection takes precedence over the selected signatures (if any signatures are selected at the same time, they will be simply ignored). To apply the operation only on the selected signatures, make sure no group is selected.
  • New option to "export to ENVI Spectral Library". (under testing) .
2.0r6 (10/24/02)
  •  Added the following Savitzky-Golay filters: "2-25-25", "2-20-20", "2-15-15", "2-10-10", "2-5-5".
  • "Jump correction" operation can now be additive or multiplicative.
  • Now, the user is required to confirm/set the 'base directory' under which SAMS stores databases and general settings. The default value continues to be either the typical home directory according to the underlying operating system (e.g., c:\documents and settings\<user login>\ on a windows system), or "Z:\" if such a root drive exists. The idea is to generalize the case where the user runs SAMS from different machines in a network and wants to access the same data through a "networked" directory (as with Z:\ in the CSTARS configuration). This occurs only once for a given machine/OS.
2.0r5 (09/26/02)
  • Status info improved. Now it shows: Signature selection, Reference signature, Group selection, and Focused element (in different text fields).
  • New explicit action Alt-Enter to choose the "reference" signature: it takes the current focused signatures as the reference.

Known bugs and limitations

  • Only some hard-coded Savitzky-Golay filters are available.
  • GER, ASD (ascii and binary), and "only-reflectance" formats are currently recognized but more testing is required here.
  • Binary ASD format is limited to: reflectance data type, and float data format. Also, binary data is assumed to be in *little-endian* format. Just tell me know if you want SAMS to read other variants.
  • Plot legends are not shown properly when the number on plotted signatures is large (because they don't fit in the display area--this is a Ptolemy limitation). Use the "Legends window" option as a workaround.
  • Plot legends are not properly updated when a signature is deselected by control-clicking.  Workaround: Press Enter, or use the "plot" option from the popup menu, to properly re-assign legends to selected signatures.
  • Drag-and-Drop has some problems depending on the Java version. But works pretty well under JRE 1.4.0_01.
  • Some user settings for printing (like landscape page orientation) don't work due to a reported Java problem, that hopefully will be fixed soon.
  • Data operations start to slow as the database gets bigger. This is because a simple persistence mechanism is currently used.  This will be fixed in a more stable version.
  • The spectra table shows you the metadata info and lets you update it, but it lacks many possible options. Let me know about any ideas!
  • Data entry validation should be improved.
